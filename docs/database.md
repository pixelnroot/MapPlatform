# docs/database.md — Database Schema & Query Patterns

> Read this before writing any SQL, any db query, or any migration.
> All table definitions, relationships, constraints, and PostGIS
> query patterns are defined here. Do not deviate from these.

---

## 1. Setup Requirements

Always enable PostGIS at the top of schema.sql before any CREATE TABLE:
```sql
CREATE EXTENSION IF NOT EXISTS postgis;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
```

All primary keys are UUID generated by `uuid_generate_v4()`.
All timestamps are `TIMESTAMPTZ` (timezone-aware).
SRID is always **4326** (WGS84) for all geometry columns.

## 2. Table Definitions

### 2.1 regions
Stores every geographic unit — world, country, division, district,
city, area — in one self-referencing table.

```sql
CREATE TABLE regions (
  id           UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name         VARCHAR(255) NOT NULL,
  name_bn      VARCHAR(255),                        -- Bengali name (optional)
  type         VARCHAR(50) NOT NULL
                 CHECK (type IN (
                   'world','country','division',
                   'district','city','area'
                 )),
  parent_id    UUID REFERENCES regions(id)
                 ON DELETE SET NULL,                -- null = root (World)
  country_code CHAR(2),                             -- ISO 3166-1 alpha-2 e.g. BD, US
  bbox_sw_lat  FLOAT,                               -- bounding box south-west lat
  bbox_sw_lng  FLOAT,                               -- bounding box south-west lng
  bbox_ne_lat  FLOAT,                               -- bounding box north-east lat
  bbox_ne_lng  FLOAT,                               -- bounding box north-east lng
  boundary     GEOMETRY(POLYGON, 4326),             -- optional polygon boundary
  osm_id       BIGINT UNIQUE,                       -- if region came from OSM
  created_at   TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_regions_parent_id  ON regions(parent_id);
CREATE INDEX idx_regions_type       ON regions(type);
CREATE INDEX idx_regions_boundary   ON regions USING GIST(boundary);
```

**Notes:**
- `parent_id = NULL` only for the single World root record
- `bbox_*` columns store the viewable bounding box for the region
  so the frontend can fly the map to that region on selection
- `boundary` is optional for MVP — can be populated later

### 2.2 categories
Place categories with display colors and icons.

```sql
CREATE TABLE categories (
  id         UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name       VARCHAR(100) NOT NULL UNIQUE,
  icon       VARCHAR(100),                          -- icon name or emoji
  color      VARCHAR(7) NOT NULL DEFAULT '#94A3B8', -- hex color for map pin
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 2.3 places
Every point of interest — shops, landmarks, hospitals, schools, etc.

```sql
CREATE TABLE places (
  id             UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name           VARCHAR(255) NOT NULL,
  name_bn        VARCHAR(255),                      -- Bengali name (optional)
  category_id    UUID REFERENCES categories(id)
                   ON DELETE SET NULL,
  region_id      UUID NOT NULL REFERENCES regions(id)
                   ON DELETE RESTRICT,
  lat            FLOAT NOT NULL,
  lng            FLOAT NOT NULL,
  geometry       GEOMETRY(POINT, 4326) NOT NULL,    -- spatial index on this
  phone          VARCHAR(50),
  opening_hours  VARCHAR(255),
  floor_details  TEXT,
  custom_notes   TEXT,
  address        TEXT,
  website        VARCHAR(500),
  source         VARCHAR(10) NOT NULL DEFAULT 'manual'
                   CHECK (source IN ('osm', 'manual')),
  osm_id         BIGINT UNIQUE,                     -- null for manual entries
  is_verified    BOOLEAN NOT NULL DEFAULT FALSE,
  created_at     TIMESTAMPTZ DEFAULT NOW(),
  updated_at     TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_places_geometry    ON places USING GIST(geometry);
CREATE INDEX idx_places_region_id   ON places(region_id);
CREATE INDEX idx_places_category_id ON places(category_id);
CREATE INDEX idx_places_source      ON places(source);
CREATE INDEX idx_places_name        ON places USING GIN(to_tsvector('english', name));
```

**Notes:**
- `geometry` is always set from lat/lng: `ST_SetSRID(ST_MakePoint(lng, lat), 4326)`
- `osm_id` is UNIQUE — prevents duplicate OSM imports
- `is_verified = TRUE` for all manually added records
- `updated_at` must be updated on every PUT via trigger (see section 4)

### 2.4 roads
Road and path geometries imported from OSM.

```sql
CREATE TABLE roads (
  id         UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name       VARCHAR(255),
  name_bn    VARCHAR(255),
  type       VARCHAR(100),                          -- OSM highway tag value
                                                   -- e.g. primary, secondary,
                                                   -- residential, footway
  region_id  UUID NOT NULL REFERENCES regions(id)
               ON DELETE RESTRICT,
  geometry   GEOMETRY(LINESTRING, 4326) NOT NULL,
  source     VARCHAR(10) NOT NULL DEFAULT 'osm'
               CHECK (source IN ('osm', 'manual')),
  osm_id     BIGINT UNIQUE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_roads_geometry  ON roads USING GIST(geometry);
CREATE INDEX idx_roads_region_id ON roads(region_id);
CREATE INDEX idx_roads_type      ON roads(type);
```

### 2.5 photos
Photos linked to a place. Stored as filenames served from /uploads.

```sql
CREATE TABLE photos (
  id         UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  place_id   UUID NOT NULL REFERENCES places(id)
               ON DELETE CASCADE,
  filename   VARCHAR(500) NOT NULL,                -- stored filename (UUID.ext)
  caption    VARCHAR(500),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_photos_place_id ON photos(place_id);
```

## 3. Trigger — Auto-update updated_at

```sql
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER update_places_updated_at
  BEFORE UPDATE ON places
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at_column();
```

## 4. PostGIS Query Patterns

These are the exact query patterns to use throughout this project.
Copy and adapt these — do not invent new spatial query patterns.

### 4.1 Insert a place with geometry
```sql
INSERT INTO places (
  id, name, category_id, region_id,
  lat, lng, geometry,
  phone, opening_hours, floor_details, custom_notes,
  source, osm_id, is_verified
) VALUES (
  uuid_generate_v4(), $1, $2, $3,
  $4, $5, ST_SetSRID(ST_MakePoint($6, $5), 4326),
  $7, $8, $9, $10,
  $11, $12, $13
)
-- Note: ST_MakePoint takes (lng, lat) — longitude first, latitude second
-- $5 = lat, $6 = lng
```

### 4.2 Fetch places inside a bounding box
```sql
-- bbox param arrives as "minLat,minLng,maxLat,maxLng"
-- Parse in JS: const [minLat, minLng, maxLat, maxLng] = bbox.split(',').map(Number)

SELECT
  p.id, p.name, p.lat, p.lng,
  p.phone, p.opening_hours, p.address,
  p.source, p.is_verified,
  c.name AS category_name,
  c.color AS category_color,
  c.icon AS category_icon
FROM places p
LEFT JOIN categories c ON c.id = p.category_id
WHERE ST_Within(
  p.geometry,
  ST_MakeEnvelope($1, $2, $3, $4, 4326)
  -- ST_MakeEnvelope(minLng, minLat, maxLng, maxLat, srid)
  -- NOTE: MakeEnvelope takes lng first, lat second (x, y order)
)
ORDER BY p.name
LIMIT $5;
```

### 4.3 Fetch roads inside a bounding box
```sql
SELECT
  id, name, type,
  ST_AsGeoJSON(geometry)::json AS geojson
FROM roads
WHERE ST_Intersects(
  geometry,
  ST_MakeEnvelope($1, $2, $3, $4, 4326)
  -- same order: minLng, minLat, maxLng, maxLat
)
LIMIT $5;
```

### 4.4 Full text search on place name
```sql
SELECT
  p.id, p.name, p.lat, p.lng,
  c.name AS category_name,
  c.color AS category_color,
  r.name AS region_name
FROM places p
LEFT JOIN categories c ON c.id = p.category_id
LEFT JOIN regions r ON r.id = p.region_id
WHERE p.name ILIKE $1
-- $1 = '%searchterm%'
AND ($2::uuid IS NULL OR p.region_id = $2)
ORDER BY p.name
LIMIT 20;
```

### 4.5 Get single place with photos
```sql
SELECT
  p.*,
  c.name AS category_name,
  c.color AS category_color,
  c.icon AS category_icon,
  r.name AS region_name,
  r.type AS region_type,
  COALESCE(
    json_agg(
      json_build_object(
        'id', ph.id,
        'filename', ph.filename,
        'caption', ph.caption
      )
    ) FILTER (WHERE ph.id IS NOT NULL),
    '[]'
  ) AS photos
FROM places p
LEFT JOIN categories c ON c.id = p.category_id
LEFT JOIN regions r ON r.id = p.region_id
LEFT JOIN photos ph ON ph.place_id = p.id
WHERE p.id = $1
GROUP BY p.id, c.name, c.color, c.icon, r.name, r.type;
```

### 4.6 Get regions as nested tree
```sql
-- Fetch all regions, build tree in JavaScript
SELECT
  id, name, name_bn, type, parent_id,
  country_code,
  bbox_sw_lat, bbox_sw_lng, bbox_ne_lat, bbox_ne_lng
FROM regions
ORDER BY type, name;
-- Build parent-child nesting in JS after fetching flat list
```

### 4.7 OSM import — idempotent insert for places
```sql
INSERT INTO places (
  id, name, category_id, region_id,
  lat, lng, geometry,
  source, osm_id, is_verified
) VALUES (
  uuid_generate_v4(), $1, $2, $3,
  $4, $5, ST_SetSRID(ST_MakePoint($6, $4), 4326),
  'osm', $7, FALSE
)
ON CONFLICT (osm_id) DO NOTHING;
-- Never updates existing OSM records — skip silently
```

### 4.8 OSM import — idempotent insert for roads
```sql
INSERT INTO roads (
  id, name, type, region_id, geometry, source, osm_id
) VALUES (
  uuid_generate_v4(), $1, $2, $3,
  ST_SetSRID(ST_GeomFromGeoJSON($4), 4326),
  'osm', $5
)
ON CONFLICT (osm_id) DO NOTHING;
```

## 5. DB Connection (backend/src/db/index.js)

```js
const { Pool } = require('pg')
require('dotenv').config()

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
})

pool.on('error', (err) => {
  console.error('Unexpected DB error', err)
  process.exit(-1)
})

module.exports = {
  query: (text, params) => pool.query(text, params),
  pool,
}
```

Usage in routes:
```js
const db = require('../db')
const result = await db.query('SELECT * FROM places WHERE id = $1', [id])
```

## 6. Critical Coordinate Order Rules

This is the most common source of bugs. Memorize this:

| Context | Order |
|---|---|
| Our database columns | `lat, lng` |
| Our API bbox param | `minLat, minLng, maxLat, maxLng` |
| ST_MakePoint() | `ST_MakePoint(lng, lat)` ← lng FIRST |
| ST_MakeEnvelope() | `ST_MakeEnvelope(minLng, minLat, maxLng, maxLat)` ← lng FIRST |
| Leaflet map center | `[lat, lng]` ← lat FIRST |
| GeoJSON coordinates | `[lng, lat]` ← lng FIRST |
| Overpass API bbox | `south, west, north, east` = `minLat, minLng, maxLat, maxLng` |

**Rule:** PostGIS and GeoJSON use X,Y = lng,lat. Leaflet uses lat,lng.
Always double-check which context you are in before passing coordinates.
